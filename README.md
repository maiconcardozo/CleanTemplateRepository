## üìã About

A production-ready .NET 8.0 authentication service implementing JWT token-based authentication and comprehensive Role-Based Access Control (RBAC). Built with Clean Architecture principles, it provides secure user management with Argon2 password hashing and a complete RESTful API.

### ‚ú® Key Features

- üîë **JWT Authentication** - Secure token-based authentication
- üë• **User Management** - Complete account management with duplicate prevention
- üõ°Ô∏è **RBAC System** - Fine-grained permissions with Claims, Actions, and mappings
- üîí **Argon2 Security** - Industry-standard password hashing
- üåê **RESTful API** - Complete CRUD operations with proper HTTP status codes
- üåç **Internationalization** - English and Portuguese support
- ‚úÖ **Validation** - FluentValidation for input validation
- üìñ **Swagger/OpenAPI** - Interactive API documentation

## üì¨ Postman Collection

**Quick Start with Postman**: A complete ready-to-use Postman collection is available with all endpoints organized by controller, including automatic token management and example payloads.

- **[Download Collection](docs/api/Authentication_API.postman_collection.json)** - Import directly into Postman
- **[Collection Guide](docs/api/POSTMAN_COLLECTION.md)** - Complete usage guide

**The collection includes:**
- ‚úÖ All 40+ API endpoints organized by controller
- ‚úÖ Automatic JWT token extraction and injection
- ‚úÖ Pre-configured request bodies with examples
- ‚úÖ Environment variable support for different deployments

**Quick setup:**
1. Download the [Postman collection](docs/api/Authentication_API.postman_collection.json)
2. Import into Postman (Import button ‚Üí Select file)
3. Run **Authentication > Generate Token** first
4. Token is automatically saved and used in all subsequent requests

**Collection structure:**
- üîê **Authentication** - JWT token generation
- üë§ **Account** - User account management
- üè∑Ô∏è **Claim** - Permission claims management
- ‚ö° **Action** - Available actions/permissions
- üîó **ClaimAction** - Claim-to-Action associations
- üë• **AccountClaimAction** - User permissions
- üè¢ **Application** - Multi-tenant application management
- üîê **ApplicationClaim** - Application-to-Claim associations

**Need help?** Check the [full Postman guide](docs/api/POSTMAN_COLLECTION.md) for detailed instructions, examples, and troubleshooting.
